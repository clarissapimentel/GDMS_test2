Use DynamicDiscovery

Let's look at the allele values for mapped objects in the same area as the QTL.

    • Click and drag the mouse cursor over the objects in the same area as the QTL
    • Press the right mouse button while the mouse cursor is in the map

The DynamicDiscovery menu appears.

    • Select Get allele values for markers/germplasms from Iris

After a pause, the DynamicDiscovery dialog appears.

    • Select Display data in a matrix
    • Select Execute

A Matrix Viewer window opens with the allele values in it.

Now, let's look at the ancestry of some of these germplasms.

    • Press and hold <Control> as you select several germplasm cells
    • Press the right mouse button

The DynamicDiscovery menu appears again.

    • Select Iris Get Ancestry

A message appears.

    • Select OK

After a pause, the DynamicDiscovery dialog appears.

    • Select ISYS Graph Viewer
    • Select Execute

The ISYS Graph Viewer appears with a hierarchical view of the requested ancestry.

To see the details of the ancestry, we'll have to use the zoom at the bottom of the window.

    • Click and drag the pointer on the Zoom scale to about 100
    • Click and drag the sliders in the scroll bars until you can see the nodes and labels
    • Select a node (not one of the nodes at the bottom of the graph)
    • The node and its label are highlighted

Notice that the cells in the Matrix Viewer are deselected and RG345 is deselected in the Map Viewer.

    • Scroll so that you can see the bottom row of nodes
    • Select one of the nodes in the bottom row

Notice that cells in the Matrix Viewer are now selected and objects are selected in the Map Viewer. When you use DynamicDiscovery, ISYS components are synchronized.

When you're finished looking at the ancestry

    • Select Quit

The Graph Viewer window closes.

    • In the Matrix Viewer window, select File >> Exit
    • Select the title bar of the Comparative Map Viewer window to bring it to the front

Let's use marker RG345 for more exploration.

    • Point the mouse cursor to RG345 as you press the right mouse button

The DynamicDiscovery menu appears.

We want to search for additional data on this marker.

    • Select Start web-based query of NCBI's Entrez using given attributes

A web browser is launched and accesses PubMed.

Our selected marker data already appears in the search line.

    • Select the Search button
    • Select Nucleotide
    • Select Go

After some time, the search results appear.

All data items that can be used as input to DynamicDiscovery are highlighted with a background color. Also, the steps to access DynamicDiscovery from a Web browser is slightly different than from an ISYS component.

Scroll through the search results. You'll see that most of them refer to our marker but only one, AQ074184, appears to represent the sequence of the actual marker.

    • Point to AQ074184 while you press <Control> and the left mouse button simultaneously

The DynamicDiscovery menu appears.

    • Select Retrieve from Entrez: Retrieve from Entrez (annotated): AQ074184
    • Select Execute

After a pause, a new DynamicDiscovery window appears.

    • Select NCGR/CGIAR Comparative Map Viewer
    • Select Execute

A map of the sequence appears in the desktop.

Make sure that only this new map is selected.

    • In the map, press your right mouse button
    • In the DynamicDiscovery menu, select NCBI BLAST

The Similarity Search Launcher window appears.

    • In the lower pane, select the box to the left of nr
    • Select Options

The Advanced Options window appears.

    • For Expectation Value, type in .001
    • Select Use These Values
    • Select Launch Query

Once the query has started, you can close the Similarity Search Launcher window.

    • Select File >> Exit

After a while, BLAST returns results in the Similarity Search Browser window.

    • Select the first record in the list
    • Press the right mouse button

The DynamicDiscovery menu appears.

    • Select Retrieve from Entrez: Retrieve from Entrez annotated: AP003222.3

A DynamicDiscovery window appears.

    • Select NCGR/CGIAR Comparative Map Viewer (you may need to scroll down in the list of Viewers to find it)
    • Select Execute

An additional marker is added to the map of the sequence we BLASTed and a new Map window appears with the annotated sequence in it.

To make the objects more distinct in the maps, we'll change the graphical properties rules.

    • Select View >> Load IsysAttribute-based property assignments from file

The Open window appears.

    • Select file graphical_properties.sample
    • Select Open
    • Select View

A new option has been added to the menu.

    • Select the box to the left of From file graphical_properties.sample: Assign colors based on Type attribute

The maps are changed to reflect the new property assignments.

So that we can see the graphical representation of all of the objects, we'll turn off the labels in the maps.

    • Make sure both maps are selected
    • Select View >> Customize layout

The Layout Dialog window appears.

    • Uncheck all of the label attributes
    • Select Apply to selected
    • Select OK

The labels are removed from the maps.

Now, let's combine the annotated sequence (15289837) with the one we selected from the BLAST return (AP003222.3).

    • Select 15289837 and AP003222.3 in the Maps Currently Available pane

    • Select View together

A combined view appears. Turn off the labels for this map.

    • Select View >> Customize layout
    • Uncheck the label attributes
    • Make sure the Combined view map is selected
    • Select Apply to selected
    • Select OK

Now, let's use a comparison to find the BLAST return objects in the Combined view.

    • Make sure 3435394 is to the right of the Combined view
    • Select the Combined view
    • Press and hold <Control> as you select 3435394
    • Select Map comparison >> View all comparisons between selected maps

The comparison shows you where the BLAST objects are in the Combined view.

    • Select the objects that are next to the BLAST return
    • Press the right mouse key
    • Select Display in table

A Table Viewer window appears with data about the two objects.

Notice that the region of AP003222.3 that matches the sequence of the marker is in the same region as a gene. Since this marker appeared to be in the most significant region of the QTL, it seems to be a reasonable hypothesis that this gene (or perhaps some other in the region) may have something to do with the trait in question. You could proceed from this point by selecting the CDS and retrieving the protein sequence from GenBank. Then you could do a BLAST against Swiss-Prot to find homologous proteins that may be better annotated in terms of their functional significance or have catalytic functions that could be studied in the context of a metabolic or regulatory pathway.
