Example Two

This example also takes us from a genetic map onto a physical map, but using a slightly different approach. In this case, we establish a connection between the maps by finding annotation on the physical map whose relevance to a marker on our map can be established directly via one of the comparison algorithms available to us. We omit many of the auxiliary paths we took in the first example, though obviously most of them are relevant in this case as well.

This example begins with the consensus map from the previous example.

    • Follow the steps in Example 1 that created the consensus map
    • Close all of the maps except the consensus map
    • Select View >> Load IsysAttribute-based property assignments from file
    • Select file graphical_properties.sample
    • Select Open
    • Select View
    • Select the box to the left of From file graphical_properties.sample: Assign colors based on Type attribute
    • Point to marker R1944 as you press the right mouse button

The DynamicDiscovery menu appears.

    • Select Start web-based query of NCBI's Entrez using given attributes

A web browser is launched and accesses PubMed.

Our selected marker data already appears in the search line.

    • Select the Search button
    • Select Nucleotide
    • Select Go

After some time, the search results appear. One of the items is AP002486.

    • To follow the hyperlink to the full record, select AP002486

The full record for the item appears. We want to use Netscape Find to see where our original marker is referenced.

    • Press and hold <Alt> as you press <f>

The Netscape: Find dialog appears.

    • Type in R1944

    • Select Find

The location of R1944 is highlighted.

    • In the Netscape: Find dialog, select Close
    • Click and drag the right-hand scroll bar to the bottom

We want to use the entire record in DynamicDiscovery. Highlighted slashes at the bottom of the record will let us do that:

    • Point to the slashes as you press <Control> and press the left mouse button

The DynamicDiscovery menu appears.

    • Select NCGR/CGIAR Comparative Map Viewer
    • Select Execute

A new map appears in the Viewer pane.

    • Select View >> Customize layout

The Layout Dialog window appears. We want to turn off the labels for now.

    • Make sure all of the label options are unchecked
    • Select OK

The labels are removed from the map.

Now, to get an idea of where our original marker is located in the larger sequence we retrieved from Entrez, we'll compare the two sequences. We want to use the pairwise substring matching algorithm.

    • Select Map comparison >> Set correspondence algorithm
    • Select the circle to the left of Pairwise substring matching
    • Select OK
    • Make sure String representation is selected
    • Select OK
    • Press and hold <Control> as you select the title bars of both maps
    • Select Map comparison >> Generate map comparison for selected maps

The comparison is illustrated in the Viewer pane.
