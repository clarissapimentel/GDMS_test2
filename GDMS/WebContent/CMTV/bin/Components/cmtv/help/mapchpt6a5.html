Defining Your Own Comparison Service

You can have the Viewer base map comparisons on your own definition of correspondence. You can define the correspondence in two ways:

  • Define and load a text file that defines the correspondences

    Each line of the text file must contain equivalent objects separated by tabs. The file can have multiple lines, as in this example:

The Load option on the Comparison Services menu lets you load the file and make it a comparison option.

  • Select corresponding objects in the Desktop and define them as equivalent

    You can accomplish this using an option on the DynamicDiscovery menu.

To load your comparison service from a file

    • Select Map comparison >> Set correspondence algorithm

The Comparison Services window appears.

    • Select Load

The Open window appears.

For help with the navigation options in this window, see Loading Map Data from a Flatfile.

    • Select your file
    • Select Open

The file is added as an option in the Comparison Services window

You can add multiple services in this way. These services remain in the Comparison Services list until you exit the Viewer.

To use your service for comparisons, while the Comparison Services window is open

    • Select the circle to the left of the algorithm you want
    • Select OK

The Choose String Aspects window appears.

This window has a list of object representations you can choose. It also lets you choose to ignore case and/or non-alphanumeric characters in the comparison. Your choices here define the content of the strings that will be used for comparison.

    • Select one or more object representations in the list

      Note: To select more than one object representation, press and hold <Control> while you select the object representations.

    • If you want the comparison to ignore case, ensure the box to the left of Make case insensitive is checked

The checkboxes are either checked (on) or unchecked (off). Simply select the box to check or uncheck it.

    • If you want the comparison to ignore non-alphanumeric characters, ensure the box to the left of Ignore non-alphanumeric is checked
    • Select OK

Your correspondence definition will be used for all comparisons in this session until you change your selection.

To define corresponding objects in the desktop

    • Ensure that all maps, comparisons, and objects are deselected before you start

Everything that is selected will be saved as part of your correspondence.

    • Select equivalent objects in multiple maps
    • In the Desktop (not within any map) press the right mouse button

The DynamicDiscovery menu appears.

The contents of this menu will vary, depending on your system.

    • Select Add selected mapped objects as correspondence to map comparison model

The first time you do this in your session, an Input dialog appears.

    • Type in the name you want for the comparison
    • Select OK

This name appears in the Map Comparisons pane listing and correspondence lines are drawn in the Viewer pane.

If you want to add more correspondences to the comparison

    • Ensure that all maps, comparisons, and objects are deselected before you start
    • Select corresponding objects in the maps
    • In the Desktop, press the right mouse button
    • Select Add selected mapped objects as correspondence to map comparison model

Any correspondences you add in this way are added to your comparison. You can define only one comparison like this per session.
