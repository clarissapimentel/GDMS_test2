Build a Consensus Map

Now, we want to build a consensus map using the second comparison.

    • Select the third comparison in the Map Comparisons pane
    • Press and hold <Control> as you select the first comparison in the pane
    • Select Delete
    • Select Yes

You can build a consensus map from the tabular view.

    • Select the one remaining comparison in the pane
    • Select View tabular

The Map Comparisons window appears.

The table shows the corresponding markers in the maps.

    • Select Consensus mapping tools

The first step in building a consensus map is to select anchor markers.

    • Select Select anchor markers >> Select well-ordered markers common to all input maps

The Viewer selects anchors and highlights them on the graphic maps.

    • Select Consensus mapping tools >> Create consensus map with selected markers as anchors

A consensus map appears in the Viewer pane.

You'll have to move the map windows around in the desktop to see all of them.

Now you can have the other markers added to the consensus map.

    • In the Map window for the consensus map, select Marker manipulations >> Position non-anchor markers

The markers are added to the consensus window.

Next, we have some QTL data we want to view in conjunction with our consensus map. To keep the desktop from being too cluttered, let's close the maps that are there now.

    • Select File >> Clear desktop
    • Select File >> Load >> Load from QTL file
    • Select file qtldata
    • Select Open
    • In the Maps Currently Available pane, press and hold <Control> as you select the consensus map and Chm 1
    • Select View together

A combined map appears in the Viewer pane. You may have to adjust the Map window's size to see the entire map. To do this

    • Move the mouse cursor to the lower right corner of the Map window
    • When the cursor changes to a right angle with an arrow pointing to it, click and drag the corner until the window is the size you want

Note that the creation of a combined map view is conceptually different than creation of a consensus map. When you combine maps, all of the data from the maps are presented visually as if they are referring to a uniform coordinate system. Combined maps are only meaningful if you choose maps whose individual coordinate systems are more or less in agreement. Also, combined maps are virtual entities and do not get represented as a new map anywhere except the Viewer pane. The consensus map, on the other hand, is a new concrete map with copies of the markers from the input maps. Resolution of the coordinate systems of the input maps to that of the consensus map is done algorithmically using interpolation based on the positions of non-anchor markers with respect to the anchor markers. This interpolation requires a comparison between a set of maps, whereas a combined map does not require marker correspondences. In this example, we cannot create a consensus that includes the QTL data because the QTL data file provides no marker information to compare with a genetic map.
