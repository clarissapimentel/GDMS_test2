Selecting a Correspondence Algorithm for Your Comparisons

Comparisons in the Viewer are made based on an algorithm. You can select from five algorithms for comparison:

  • Compare objects based on IsysObject equivalency

    All objects in ISYS have a definition of what's equivalent to them (e.g., if the name is exactly the same, the objects are equivalent; if some other attribute is the same, the objects are equivalent). This definition comes from the data source and is the same criteria used for synchronization in ISYS.

    This is the only algorithm that doesn't ask you how to convert the data into strings for comparison.

  • Create sets based on common substrings

    This algorithm creates sets of objects that have substrings in common.

  • Hash into equivalence sets

    This algorithm creates sets of equivalent objects.

  • Pairwise string comparisons using Levenstein distance

    This algorithm create pairs of objects containing matching strings. It determines equivalence based on a fuzzy model so it may result in false positives.

  • Pairwise substring match

    This algorithm creates pairs of objects containing matching substrings.

You can also define your own comparison criteria. For more information about this, see Defining Your Own Comparison Service.

To select a correspondence algorithm

    • Select Map comparison >> Set correspondence algorithm

The Comparison Services window appears.

Hash into equivalence sets is the default algorithm.

    • Select the circle to the left of the algorithm you want
    • Select OK

If you selected to compare based on IsysObject equivalency, your selection is complete. Otherwise, the Choose String Aspects window appears.

This window has a list of object representations you can choose. Your choices here define the content of the strings that will be used for comparison. The top option, String representation, results in the entire record being compared. Other options let you select to compare only particular objects. Two checkbox options near the bottom of the window let you choose to ignore case and/or non-alphanumeric characters in the comparison.

    • Select one or more object representations in the list

      Note: To select more than one object representation, press and hold <Control> while you select the object representations.

    • If you want the comparison to ignore case, ensure the box to the left of Make case insensitive is checked

The checkboxes are either checked (on) or unchecked (off). Simply select the box to check or uncheck it.

    • If you want the comparison to ignore non-alphanumeric characters, ensure the box to the left of Ignore non-alphanumeric is checked
    • Select OK

The algorithm and string aspects you selected will be used for all comparisons in this session until you change your selection.
