Visualization of "raw" QTL Data

Many data providers present QTL data as regions on maps whose likelihood ratios or LOD scores are above some threshold value. The tool can display this type of data similarly to the display of other interval-based map displays. Additionally, there is value to be gained by presenting the complete distribution of scores for a QTL across the entire map; sometimes, regions that do not meet a strictly defined threshold can still lend weight to the assessment of a region's significance for a certain trait. With this perspective in mind, and mindful of the importance of other sorts of numerical distributions such as average GC content in sequence data, we extended the tool to be able to visualize and manipulate data of a more continuous nature.

By default, traits are displayed using a graphical representation that maps numerical values to colors. The layout is specified by the value compmapviewer.continuous_distribution_color_map, and declared in the configuration file, compmapviewer.prop.

This file simply specifies ranges of numerical values and the color ranges to which they should be mapped (colors may be specified either using the set of "named values" defined in java.awt.Color as static instances, or as RGB value triples as in the file qtl_color_map2). This file can be edited to represent a new mapping, if desired. After the file has been edited, the mapping can be reloaded dynamically. From the main menu:

    • Select View >> Reload color map>> For continuous distributions.

Another visualization that is useful for this data is a standard histogram display. This can be enabled by bringing up the Layout Dialog from the main menu:

    • Select View>> Customize layout
In the tab labeled Panel 3 ("Continuous distributions"):
    • Select Create graph renderings of distributions
See Customize Layout for details on using the Layout Dialog.

The same two visualizations can also be used to display aggregate representations of sets of trait data. In a map frame's View menu, you will see two checkbox items: Show aggregate graph distribution and Show aggregate color distribution. When these are checked and trait data has been loaded into the tool, the last graphic displayed in the frame (furthest from the ruler that represents the coordinates of the map) represents dynamically computed averages of the set of traits that are present in the visualization at any given time.

Graphical attributes displayed on the maps can be modified based on the attributes of the underlying data (see Defining the Graphical Properties of Your Maps In the same manner, traits have some graphical properties that can be modified in this way. As an example, you can map the phenotypic trait name to a "scaling factor", which modifies the intensity of the trait display in the singular trait representations, and weights the contribution of the trait to the average value in the aggregate distributions. To see this, load some traits onto one of the marker maps (as described in Combinable maps). From the main menu:

    • Select View >> Load IsysAttribute-based property assignments from file
    • Select trait_scaling_factors
    • Go back to View
    • Select From file trait_scaling_factors...
You should see some of the traits pale, corresponding to the scaling factor corresponding to the trait in the file. In addition, the contribution of each trait to the aggregate distribution should have changed in accordance with these values.

In order to help visualize large sets of traits, it is also possible to adjust the width of the graphical representations. From the main menu:

    • Select View >> Set widget width
and follow the instructions for Set Color/Font for Selected; to apply the change to all traits, see Set Color/Font for User-Defined Rule and choose Continuous Distributions.

Some other features that are useful for manipulating these QTL distributions are:

Sorting by value

    • Right-click on any of the traits, and you should see a menu option to sort the traits by the value of the distribution at that position (note that although you will see the same option when right-clicking on the aggregate, at this time the sorting will not be applied to the simple traits in this case).

Selecting by threshold

    • Right-click in a frame that contains trait data, and you should see an option to select traits by threshold. Choosing this menu item will bring up the threshold filter dialog described elsewhere, and the traits meeting the criteria specified will be selected.

Removing from view

    • Traits that have been added to a view (see Combinable Maps) can be removed by selecting them using the filter threshold and choosing Edit >> Remove selected mapped objects. You can also do this to data that has been originally loaded as part of the map.
