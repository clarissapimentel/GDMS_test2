Compare Maps

In this Map window, select Map comparison >> Find comparable maps

The Map Comparisons dialog appears.

The default here is "2".

    • Select OK to accept the default

After a pause, a second map appears. You will need to click and drag one of the maps in the desktop so you can see both of them.

    • In the Map window header bar, click and drag the window

Lines connect the corresponding markers in the maps. You can see that there are three corresponding sets of markers that are "equivalent" because their marker names match.

This comparison was performed using the default comparison service: Hash into equivalent sets. For information on the comparison services, see Selecting a Correspondence Algorithm for Your Comparisons. Let's try another service and see what kind of results we get.

    • In the Map Viewer window, select Map comparison >> Set correspondence algorithm

The Comparison Services window appears.

    • Select the circle for Pairwise substring matching

    • Select OK

The Choose String Aspects window appears.

    • Select OK to select the default selection

Make sure both maps are selected. Selected maps have blue title bars.

    • Press and hold <Control> as you select the title bar of both maps
    • Select Map comparison >> Generate map comparison for selected maps

This comparison includes one additional correspondence, a match between markers RG532 and RG532B. This is likely a significant match considering the agreement in order with respect to the other matches.

Let's try one more comparison service.

    • Select Map comparison >> Set correspondence algorithm
    • In the Comparison Services window, select the circle to the left of Pairwise string comparisons using Levenstein distance

For information on the comparison services, see Selecting a Correspondence Algorithm for Your Comparisons.

    • Select OK
    • Select OK again
    • Make sure both maps are selected
    • Select Map comparison >> Generate map comparison for selected maps

The correspondences are illustrated on the maps.

This comparison has created many correspondences. In this case it looks unlikely, though, that most of the correspondences represent anything meaningful. Of the three comparisons, the second seems to contain the greatest number of significant matches. So, we will continue to work with that comparison.

Each of the three comparisons is listed in the Map Comparisons pane.

We could try other comparison services. In addition to the services available in the Viewer, developers can add comparison services to ISYS. These services would then be available via ISYS's service programs.
